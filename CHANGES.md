# Добавлена функциональность периодичности задач

## Что сделано

В API добавлена возможность задавать настройки повторяемости при создании и редактировании задач.

### Изменения в базе данных

Таблица `tasks` расширена двумя полями:
- `scheduled_at` — срок выполнения задачи или якорная дата для ежедневного повтора
- `recurrence` — настройки повторяемости, хранятся в формате JSON

### Новый API endpoint

`GET /api/v1/tasks/{id}/occurrences?from=YYYY-MM-DD&to=YYYY-MM-DD`

Возвращает список дат, в которые задача должна быть выполнена в указанном диапазоне. Если `from`/`to` не указаны — диапазон по умолчанию: сегодня + 30 дней.

### Поддерживаемые типы повторяемости

| Тип | Описание |
|---|---|
| `daily` | Каждые N дней, начиная с `scheduled_at` |
| `monthly` | Каждое N-е число месяца (1–30) |
| `dates` | Только на конкретные даты |
| `even_odd` | Все чётные или нечётные числа месяца |

### Покрытие тестами

Добавлено 22 unit-теста для логики вычисления дат: все 4 типа повторяемости, граничные случаи (февраль без 30-го числа, пустой диапазон, некорректные форматы).

## Инструкция пользователя

### Запуск

```bash
docker compose down -v
docker compose up --build
```

> `down -v` нужен при первом запуске после обновления — он пересоздаёт базу данных с новой схемой.

После запуска:
- API: `http://localhost:8080/api/v1`
- Swagger UI: `http://localhost:8080/swagger/`

---

### Создать разовую задачу (без повторов)

```http
POST /api/v1/tasks
Content-Type: application/json

{
  "title": "Консультация пациента",
  "description": "Иванов И.И., палата 5",
  "status": "new",
  "scheduled_at": "2026-04-15T10:00:00Z"
}
```

---

### Создать задачу с повторяемостью

**Ежедневно** (каждый день, начиная с 1 апреля):

```http
POST /api/v1/tasks
Content-Type: application/json

{
  "title": "Обзвон пациентов",
  "scheduled_at": "2026-04-01T09:00:00Z",
  "recurrence": {
    "type": "daily",
    "interval": 1
  }
}
```

**Каждые 2 дня:**

```json
"recurrence": { "type": "daily", "interval": 2 }
```

**Каждое 5-е число месяца:**

```json
"recurrence": { "type": "monthly", "day_of_month": 5 }
```

**На конкретные даты:**

```json
"recurrence": {
  "type": "dates",
  "dates": ["2026-04-15", "2026-07-15", "2026-10-15"]
}
```

**По нечётным числам месяца:**

```json
"recurrence": { "type": "even_odd", "parity": "odd" }
```

**По чётным числам месяца:**

```json
"recurrence": { "type": "even_odd", "parity": "even" }
```

---

### Посмотреть даты выполнения задачи

```http
GET /api/v1/tasks/1/occurrences?from=2026-04-01&to=2026-04-30
```

Ответ:

```json
{
  "task_id": 1,
  "from": "2026-04-01",
  "to": "2026-04-30",
  "occurrences": [
    "2026-04-01",
    "2026-04-03",
    "2026-04-05",
    ...
  ]
}
```

Если параметры `from`/`to` не указаны, используется диапазон от сегодня до +30 дней:

```http
GET /api/v1/tasks/1/occurrences
```

---

### Обновить настройки повторяемости

```http
PUT /api/v1/tasks/1
Content-Type: application/json

{
  "title": "Обзвон пациентов",
  "status": "new",
  "scheduled_at": "2026-04-01T09:00:00Z",
  "recurrence": {
    "type": "daily",
    "interval": 3
  }
}
```

Чтобы убрать повторяемость, просто не передавайте поле `recurrence` (или передайте `null`).

---

### Ограничения

- Диапазон в `/occurrences` не может превышать **1 год**
- Максимум **366 дат** в одном ответе
- Для типа `monthly` число месяца ограничено **1–30** (если в месяце нет такого числа — он пропускается)
- Для типа `daily` поле `scheduled_at` обязательно
